## Milestone 3: Codex Workflow Parity

- Full PTY terminal sessions with job control and TUI handling, richer worktree branch lifecycle polish, local env orchestration, and richer GitHub review workflows. Local integrated-terminal commands now run through `PTYProcessSession`, which attaches a command to a `posix_openpt` master/slave pair so programs that probe `isatty()` get true terminal behavior, streams merged terminal output through the existing `ShellProcessEvent` contract with the same cancel/timeout/exit-code semantics as the pipe runner, accepts user input through the master fd, and propagates terminal window size (rows/columns) through `TIOCSWINSZ` so `stty size`/ncurses TUIs lay out correctly. Non-terminal shell tools and SSH Remote terminal commands intentionally stay pipe/noninteractive for clean tool output. Job control and full TUI redraw handling remain the next steps.
- Per-file project-instruction scopes now attach to loaded `AGENTS.md`, `.quillcode/rules.md`, and `.quillcode/instructions.md` records, flow into TrustedRouter hidden system context, and show in Activity sources. Activity also surfaces duplicate-scope, nested-override, and conservative semantic-conflict diagnostics for explicit opposing instructions over overlapping scopes, then promotes conflicts into a dedicated Instruction Review section with issue counts and toggle controls. Source-to-file jump/edit/resolve UI remains pending.
- Explicit subagent workflows now have a replayable `host.subagents.update` progress schema, a visible Activity Subagents section, a deterministic `/subagents objective | Name: role` slash command, and a real parallel scheduler that fans out injectable workers while emitting queued/running/completed/cancelled progress through the same event/tool contract. Stop All cancellation now preserves stopped state and cancelled progress for slash-command workers. Subagent workers are now model-backed by default: `LLMWorkspaceSubagentWorker` runs each job as a focused tool-free `LLMClient` turn through the scheduler's existing worker closure, while the scheduler stays injectable for cancellation tests. The scheduler now also runs a task dependency graph: `/subagents` workers can declare prerequisites with `Name after Dep1, Dep2: role`, and the scheduler fans out jobs in dependency waves, surfaces still-waiting jobs as `blocked`, skips dependents whose prerequisite failed or was cancelled, and breaks dependency cycles so a run always terminates. Dependent model-backed workers now also receive the concrete result summaries of their completed prerequisites in their prompt, so a `Verifier after Builder` turn can build on what `Builder` actually produced. Nested multi-level plans remain pending.
- Browser rendering adapter, live DOM/page inspection implementation for dynamic pages, richer browser comments, and browser-open snapshot fetching from agent-driven navigation. The live DOM capture contract now exists behind `BrowserLiveDOMCapturing`, and the macOS desktop app can upgrade fetched HTTP(S) pages through an offscreen WebKit rendered-DOM capture that reuses WebKit's persistent cookie/session data store by default. A reusable visible macOS WebKit browser-session window can now be opened from the browser pane, menu bar, or command palette to sign into sites using the same profile. Browser preview state now has focused tab records with isolated history/snapshot/comment state plus native/static HTML/Playwright tab create, switch, and close flows. The macOS visible WebKit session now consumes a shared browser-session sync snapshot, renders the current browser tab set as native tabs, and reports visible-session tab selection/title/URL changes back into the preview model through an app-layer update contract. Linux/browser-process adapters and richer live-DOM feedback loops still need implementation.
- Computer Use platform backends, app approvals, and agent wiring. macOS now has permission-aware status, screenshot/input primitives, Settings onboarding actions, and structured agent tools for screenshot/click/type/scroll/move/key dispatch through the active backend; Linux now has helper-backed screenshot/input routing for Wayland and X11 when required helpers are installed. Richer onboarding and multi-step model feedback over screenshot results remain.
- Full embedded PDF/doc/sheet/slide/appshot artifact renderers, signed remote marketplace distribution, MCP streaming beyond the generic verified-server call path, and richer prompt/resource UX. Structured PDF/document/spreadsheet/presentation/appshot artifact cards now exist in the shared tool-card surface, including `.appshot.json` bundle classification without raw JSON text-preview noise. Project extension manifests can expose explicit bounded install/update commands that run through the same shell tool-card path and refresh project metadata afterward. Project-local `.quillcode/marketplace/*.json` entries now provide a bounded marketplace browse/install lane: entries declare their kind, render as Available, run their install command through the same shell tool-card path, and disappear once the installed manifest is discovered. MCP resource reads, prompt gets, and bounded per-tool input schema summaries now run through allowlisted Ready-server agent tools and the Extensions UI, building on the tested discovery UI contract. Ready MCP resources and prompts are directly actionable from the Extensions pane and command palette, with results recorded as normal tool cards and assistant summaries. Explicit memory editing and Forget actions now cover global, local project, and SSH Remote project memory files. Richer memory redaction/Chronicle jobs, fully autonomous memory inference, conflict review, project-memory review UX, event-source monitor adapters, and real subagent execution remain pending.
