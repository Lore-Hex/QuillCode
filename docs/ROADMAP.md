## Milestone 3: Codex Workflow Parity

- Full PTY terminal sessions with job control and TUI handling, richer worktree branch lifecycle polish, local env orchestration, and richer GitHub review workflows. A pseudo-terminal foundation now exists: `PTYProcessSession` runs a command attached to a `posix_openpt` master/slave pair so programs that probe `isatty()` get true terminal behavior, streaming merged terminal output through the existing `ShellProcessEvent` contract with the same cancel/timeout/exit-code semantics as the pipe runner, propagating the terminal window size (rows/columns) to the child through `TIOCSWINSZ` so `stty size`/ncurses TUIs lay out correctly, accepting interactive keystroke input through `sendInput` so shells, REPLs, and prompts can be driven, and re-applying the window size to a running session through `resize` so `SIGWINCH`-aware programs reflow when the terminal changes. The live local workspace terminal now launches commands through that PTY path, seeds new sessions from the SwiftUI terminal pane's measured cell size, forwards pane-size changes to the active local PTY session, exposes Stop/Suspend/Resume, and renders common ANSI/cursor-addressed TUI redraws through a single-page screen buffer while SSH Remote terminal commands stay on the noninteractive SSH pipe path. The renderer now handles bounded scroll regions, reverse-index, explicit scroll up/down, insert/delete line, alternate-screen latest-frame preservation, width-two CJK/emoji/emoji-presentation advancement, ZWJ grapheme-cluster preservation, combining-mark cursor accounting, narrow-by-default ambiguous-width policy for stable transcript scrollback, automatic wide ambiguous-width policy for CJK locale terminal surfaces, and a `QUILLCODE_TERMINAL_AMBIGUOUS_WIDTH` override for support/debugging. Full curses terminal-emulator semantics, richer job control, and deeper alternate-screen history remain the next steps.
- Per-file project-instruction scopes now attach to loaded `AGENTS.md`, `.quillcode/rules.md`, and `.quillcode/instructions.md` records, flow into TrustedRouter hidden system context, and show in Activity sources. Activity also surfaces duplicate-scope, nested-overlap, explicit nested-override, and conservative semantic-conflict diagnostics for explicit opposing instructions over overlapping scopes, then promotes conflicts into a dedicated Instruction Review section with issue counts and toggle controls. Source rows expose typed Open/Edit actions, conflict diagnostics expose line-specific Open Source/Edit Source plus Resolve/Dismiss actions, Resolve drafts include precise `file:line` source targets with excerpts and suggested edits, dismissed diagnostics persist as project-scoped resolution records, and diagnostics that disappear after metadata refresh are recorded as resolved-by-edit audit records. Local desktop source actions open readable workspace files in the native editor at the exact line when possible and fall back to the model file-read/edit-draft path otherwise. Deterministic two-line semantic conflicts now expose direct keep-side patch actions that remove the opposite conflicting line through `host.apply_patch`; exact duplicate same-scope diagnostics expose direct Clear duplicate actions through `host.file.write` with explicit empty content; repeated broad lines inside nested instruction files expose direct Remove repeated lines actions through `host.apply_patch`; explicit nested override diagnostics expose direct Remove override lines actions through `host.apply_patch` when loaded excerpts still match. These action types refresh project context after success; non-identical duplicate merges and semantic nested override rewrites remain pending.
- Explicit subagent workflows now have a replayable `host.subagents.update` progress schema, a visible Activity Subagents section, a deterministic `/subagents objective | Name: role` slash command, and a real parallel scheduler that fans out injectable workers while emitting queued/running/completed/cancelled progress through the same event/tool contract. Stop All cancellation now preserves stopped state and cancelled progress for slash-command workers. Subagent workers are now model-backed by default: `LLMWorkspaceSubagentWorker` runs each job as a focused tool-free `LLMClient` turn through the scheduler's existing worker closure, while the scheduler stays injectable for cancellation tests. The scheduler now also runs a task dependency graph: `/subagents` workers can declare prerequisites with `Name after Dep1, Dep2: role`, and the scheduler fans out jobs in dependency waves, surfaces still-waiting jobs as `blocked`, skips dependents whose prerequisite failed or was cancelled, and breaks dependency cycles so a run always terminates. Dependent model-backed workers now also receive the concrete result summaries of their completed prerequisites in their prompt, so a `Verifier after Builder` turn can build on what `Builder` actually produced. The scheduler also honors an optional per-run concurrency bound (`maxConcurrentWorkers`) that caps how many ready workers fan out at once and starts another as each finishes, matching Codex's bounded parallelism; the unbounded default is unchanged. The bound is settable from the slash command with a leading `xN` token (e.g. `/subagents x2 ship release | Builder: compile | Verifier after Builder: run tests`), honored only when it is the first word and leaves a non-empty objective. Hierarchical slash paths now cover nested plan presentation and same-group dependency shorthand (e.g. `Frontend/UX` plus `Frontend/Tests after UX`). Recursive delegated subagent spawning is implemented for slash-command runs through bounded `[[DELEGATE: name | role]]` markers, parser-backed child requests, scheduler depth/total-job caps, and replayable Activity progress for nested children.
- Browser rendering adapter, live DOM/page inspection implementation for dynamic pages, richer browser comments, and browser-open snapshot fetching from agent-driven navigation. The live DOM capture contract now exists behind `BrowserLiveDOMCapturing`, and the macOS desktop app can upgrade fetched HTTP(S) pages through an offscreen WebKit rendered-DOM capture that reuses WebKit's persistent cookie/session data store by default. A reusable visible macOS WebKit browser-session window can now be opened from the browser pane, menu bar, command palette, or `/session [target]` slash command to sign into sites using the same profile. Browser preview state now has focused tab records with isolated history/snapshot/comment state plus native/static HTML/Playwright tab create, switch, and close flows. The macOS visible WebKit session now consumes a shared browser-session sync snapshot, renders the current browser tab set as native tabs, reports visible-session tab selection/title/URL changes back into the preview model through an app-layer update contract, and sends rendered live-DOM snapshots from completed visible-session navigations through the same app-layer snapshot model as offscreen captures. Linux desktop builds now have a headless Chromium/Chrome browser-process live-DOM adapter that renders HTTP(S)/file URLs with isolated profile state and bounded dumped DOM capture when an installed browser is available. Native desktop smoke now opens local HTML, preserves a browser comment, and proves the composer-driven `host.browser.inspect` path in release artifacts. Deeper bidirectional browser-session actions and Linux visible-session control still need implementation.
- Computer Use platform backends, app approvals, and agent wiring. macOS now has permission-aware status, screenshot/input primitives, Settings onboarding actions, focused-app detection, and structured agent tools for screenshot/click/type/scroll/move/key dispatch through the active backend; Linux now has helper-backed screenshot/input routing for Wayland and X11 when required helpers are installed. - Full embedded PDF/doc/sheet/slide/appshot artifact renderers, signed remote marketplace distribution, MCP streaming beyond the generic verified-server call path, and richer prompt/resource UX. Structured PDF/document/spreadsheet/presentation/appshot artifact cards now exist in the shared tool-card surface, including `.appshot.json` bundle classification without raw JSON text-preview noise. Project-local `.quillcode/marketplace/*.json` entries now provide a bounded marketplace browse/install lane: entries declare their kind, render as Available, run their install command through the same shell tool-card path, and disappear once the installed manifest is discovered. MCP resource reads, prompt gets, and bounded per-tool input schema summaries now run through allowlisted Ready-server agent tools and the Extensions UI, building on the tested discovery UI contract. Ready MCP resources and prompts are directly actionable from the Extensions pane and command palette, with results recorded as normal tool cards and assistant summaries. Explicit memory editing and Forget actions now cover global, local project, and SSH Remote project memory files, and the Memories pane now surfaces deterministic review rows for blocked credential-like memory attempts plus obvious opposing global/project memory notes with edit actions. Deterministic event-source adapters now exist through the `AutomationEventSource` protocol: `FileChangeEventSource` fires when a watched file appears or is modified after the last check, `URLLastModifiedEventSource` fires when an HTTP(S) resource reports a Last-Modified timestamp newer than the monitor's last run, and `URLFeedUpdateEventSource` fires when a bounded RSS/Atom feed reports a published or updated timestamp newer than the last run. `/monitor file|last-modified|feed ...` plus the command palette/create menu can create those persisted monitors without hand-editing automation JSON. The desktop automation tick resolves configured event sources for active event monitors and creates monitor threads with trigger context. Linux notification command planning now maps agent-run and automation notifications to bounded `notify-send` argv without shell interpolation, `LinuxNotificationCommandRunner` delivers that argv through a fake-helper-tested process boundary, the desktop notifier factory selects that Linux path at runtime, and persisted Settings controls now gate agent-run alerts, inactive-only run alerts, and automation alerts without restarting the desktop app. Deeper memory redaction/Chronicle jobs, fully autonomous memory inference, broader project-memory review UX, broader event-source adapters and their engine wiring, richer recursive-subagent UX controls, and deeper delegated-agent observability remain pending.
